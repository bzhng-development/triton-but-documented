class triton.Config
==================

Represents a possible kernel configuration for the auto-tuner to try.

This object encapsulates compilation parameters and meta-arguments used to
instantiate a GPU kernel. Instances are typically passed to
:func:`triton.autotune` to define the search space for performance tuning.

Parameters
----------
kwargs : dict
    A dictionary of meta-parameters to pass to the kernel as keyword arguments.
    Keys must match the names of `tl.constexpr` arguments in the kernel definition.
num_warps : int, optional
    The number of warps to use for the kernel. Defaults to 4.
    Each warp consists of 32 threads; for example, ``num_warps=8`` results in
    256 concurrent threads per block.
num_stages : int, optional
    The number of stages for software pipelining of loops. Defaults to 3.
    Increasing this value can improve instruction-level parallelism, particularly
    for matrix multiplication workloads on NVIDIA Ampere (SM80+) and newer GPUs.
num_ctas : int, optional
    The number of blocks in a block cluster. Defaults to 1.
    This parameter enables Cooperative Thread Arrays (CTA) and is only supported
    on NVIDIA Hopper (SM90+) GPUs.
maxnreg : int, optional
    Maximum number of registers one thread can use. Corresponds to the PTX
    ``.maxnreg`` directive. Defaults to ``None`` (compiler decides).
    Not supported on all platforms.
pre_hook : callable, optional
    A function called before the kernel is launched. It receives the full
    argument dictionary as its single input. Useful for custom setup logic
    during autotuning.
ir_override : str, optional
    Path to a file containing user-defined intermediate representation (IR)
    to override the compiler's generated code. Supported extensions include
    ``.ttgir``, ``.llir``, ``.ptx``, and ``.amdgcn``. Defaults to ``None``.

Examples
--------
Define a set of configurations to test different block sizes and warp counts:

.. code-block:: python

    import triton
    import triton.language as tl

    configs = [
        triton.Config(kwargs={'BLOCK_SIZE': 128}, num_warps=4, num_stages=3),
        triton.Config(kwargs={'BLOCK_SIZE': 1024}, num_warps=8, num_stages=4),
        triton.Config(kwargs={'BLOCK_SIZE': 64}, num_warps=2, num_stages=2),
    ]

    @triton.autotune(configs=configs, key=['n'])
    @triton.jit
    def vector_add_kernel(x_ptr, y_ptr, out_ptr, n, BLOCK_SIZE: tl.constexpr):
        # Kernel implementation
        ...

Notes
-----
- The ``num_stages`` optimization is most effective on GPUs with large shared
  memory capacity and async memory copy instructions (SM80+).
- ``num_ctas`` requires hardware support for block clusters (SM90+); setting
  this on older architectures will result in a compilation error.
- When used with :func:`triton.autotune`, each configuration causes the kernel
  to be compiled and benchmarked. Ensure side effects are handled correctly
  using ``reset_to_zero`` or ``pre_hook`` if the kernel modifies input tensors.
