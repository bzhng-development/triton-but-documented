Compile a Triton kernel source into an executable GPU kernel.

This function takes a Triton kernel definition (either as a Python function or a path to an IR file) and runs the full compilation pipeline, including parsing, optimization, code generation, and caching. It returns a ``CompiledKernel`` object that can be launched on the GPU.

Parameters
----------
src : str or ASTSource
    The source to compile. This can be:
    - A file path (string) to a Triton IR file (e.g., ``.ttir``, ``.ttgir``, ``.ptx``).
    - An ``ASTSource`` object, typically created internally when decorating a function with ``@triton.jit``.
target : GPUTarget, optional
    The target GPU architecture to compile for. If ``None``, defaults to the currently active device target.
options : dict, optional
    A dictionary of compilation options. Common keys include ``num_warps``, ``num_ctas``, and ``num_stages``.
    If ``None``, default options are used.
_env_vars : dict, optional
    Environment variables used to invalidate the cache. If ``None``, automatically detected invalidating
    environment variables are used.

Returns
-------
CompiledKernel
    An object containing the compiled binary, intermediate representations (IR), and metadata.
    The kernel can be executed by calling the returned object with a grid configuration and arguments.

Notes
-----
The compilation process is cached based on a hash of the source, target, options, and environment variables.
Subsequent calls with identical inputs will return a cached result unless ``TRITON_ALWAYS_COMPILE=1`` is set.

Intermediate representations (IR) and binaries are stored in the cache directory (controlled by ``TRITON_CACHE_DIR``).
Advanced users can override specific compilation stages by placing files in the override directory (controlled by
``TRITON_KERNEL_OVERRIDE``).

Examples
--------
.. code-block:: python

    import triton
    import triton.language as tl

    @triton.jit
    def add_kernel(
        x_ptr, y_ptr, output_ptr, n,
        BLOCK_SIZE: tl.constexpr,
    ):
        pid = tl.program_id(axis=0)
        block_start = pid * BLOCK_SIZE
        offsets = block_start + tl.arange(0, BLOCK_SIZE)
        mask = offsets < n
        x = tl.load(x_ptr + offsets, mask=mask)
        y = tl.load(y_ptr + offsets, mask=mask)
        output = x + y
        tl.store(output_ptr + offsets, output, mask=mask)

    # Compile the kernel explicitly (usually done implicitly via @triton.jit)
    compiled_kernel = triton.compile(
        src=add_kernel,
        options={"num_warps": 4, "BLOCK_SIZE": 1024}
    )

    # Access compiled metadata
    print(f"Compiled for: {compiled_kernel.metadata.target}")
    print(f"Shared memory usage: {compiled_kernel.metadata.shared} bytes")

    # The result is typically executed via the kernel runner, not directly:
    # compiled_kernel[grid](args...)
