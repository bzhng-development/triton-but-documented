triton.language.float8e5
=======================

8-bit floating point type with 5 exponent bits and 2 mantissa bits (E5M2).

This dtype corresponds to the IEEE 754 binary8 format (E5M2), commonly used in
machine learning workloads for reduced precision arithmetic. It supports
special values including NaN, infinity, and subnormal numbers.

Parameters
----------

None (dtype instance).

Returns
-------

dtype
    A ``triton.language.dtype`` instance representing the float8 E5M2 format.

Notes
-----

*   **Bit Layout**: 1 sign bit, 5 exponent bits, 2 mantissa bits.
*   **Range**: Approximately ``6.55e4`` (max finite value) to ``6.10e-5`` (min normal positive).
*   **Usage**: Typically used in ``tl.load``, ``tl.store``, or pointer type definitions
    to specify memory layout. Arithmetic operations involving ``float8e5`` usually
    require explicit casting to ``float16`` or ``float32`` before computation, as
    most GPU tensor cores do not support direct arithmetic on 8-bit floats without
    conversion.
*   **Hardware Support**: Requires GPU architectures with native support for E5M2
    (e.g., NVIDIA Hopper H100 or later via specific load/store instructions).

Examples
--------

.. code-block:: python

    import triton
    import triton.language as tl

    @triton.jit
    def kernel(ptr, BLOCK_SIZE: tl.constexpr):
        # Define a pointer type using float8e5
        f8_ptr = ptr.to(tl.pointer_type(tl.float8e5))

        # Load data as float8e5
        x = tl.load(f8_ptr + tl.arange(0, BLOCK_SIZE))

        # Cast to float16 for computation
        x_fp16 = x.to(tl.float16)

        # Perform arithmetic
        y = x_fp16 * 2.0

        # Store result back (requires appropriate hardware support)
        tl.store(f8_ptr + tl.arange(0, BLOCK_SIZE), y)
