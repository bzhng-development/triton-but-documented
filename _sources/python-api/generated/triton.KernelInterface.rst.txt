class KernelInterface
=================

.. currentmodule:: triton

.. class:: KernelInterface()

   Abstract base class for generic Triton kernel types.

   This class defines the interface for JIT-compiled kernel functions. It
   provides the mechanism to bind a launch grid to a kernel via the
   ``__getitem__`` method, enabling the standard launch syntax
   ``kernel[grid](*args, **kwargs)``.

   Parameters
   ----------
   None
      This is an abstract base class and is not instantiated directly.

   Attributes
   ----------
   run : T
      Abstract method representing the underlying execution logic. Must be
      implemented by subclasses (e.g., :class:`triton.JITFunction`).

   Methods
   -------
   warmup(*args, grid, **kwargs)
      Runs the kernel in warmup mode to trigger compilation without
      executing the full launch logic. Converts arguments to mock tensors
      if necessary.
   __getitem__(grid)
      Returns a callable proxy that memorizes the specified ``grid``.
      Calling this proxy executes the kernel with the bound grid.

   Notes
   -----
   Users typically interact with instances of :class:`triton.JITFunction`,
   which inherits from ``KernelInterface``. Direct instantiation of
   ``KernelInterface`` is not supported.

   The launch workflow involves two steps:
   1. Bind the grid: ``proxy = kernel[grid]``
   2. Execute: ``proxy(*args, **kwargs)``

   Examples
   --------
   >>> import triton
   >>> import triton.language as tl
   >>>
   >>> @triton.jit
   >>> def add_kernel(x_ptr, y_ptr, output_ptr, n):
   >>>     pid = tl.program_id(axis=0)
   >>>     if pid < n:
   >>>         tl.store(output_ptr + pid, tl.load(x_ptr + pid) + tl.load(y_ptr + pid))
   >>>
   >>> # Define the grid (e.g., 1D grid of size 1024)
   >>> grid = (1024,)
   >>>
   >>> # Launch the kernel using the KernelInterface syntax
   >>> # kernel[grid] returns a callable that executes the kernel
   >>> add_kernel[grid](x_ptr, y_ptr, output_ptr, n=1024)
