static_assert
=============

.. currentmodule:: triton.language

.. function:: static_assert(cond, msg='', _semantic=None)

   Assert the condition at compile time. Does not require that the ``TRITON_DEBUG`` environment variable is set.

   Parameters
   ----------
   cond : constexpr
      The condition to assert. Must be a compile-time constant expression.
   msg : str, optional
      The error message to display if the assertion fails.
   _semantic : internal, optional
      Internal argument used by the compiler; do not set manually.

   Returns
   -------
   None

   Notes
   -----
   This function triggers a compile-time error if ``cond`` evaluates to ``False``.
   It is useful for validating ``constexpr`` values (e.g., block sizes, power-of-two constraints)
   before kernel compilation proceeds. Unlike :func:`device_assert`, this check occurs during
   compilation and does not inject runtime instructions.

   Examples
   --------
   .. code-block:: python

      import triton
      import triton.language as tl

      @triton.jit
      def kernel(BLOCK_SIZE: tl.constexpr):
          # Ensures BLOCK_SIZE is 1024 at compile time
          tl.static_assert(BLOCK_SIZE == 1024, "BLOCK_SIZE must be 1024")
          # Kernel logic...
          pass
