static_print¶

triton.language.static_print(*values, sep: str = ' ', end: str = '\n', file=None, flush=False, _semantic=None)¶

Print the values at compile time. The parameters are the same as the builtin print.

NOTE: Calling the Python builtin print is not the same as calling this, it instead maps to device_print, which has special requirements for the arguments.

tl.static_print(f"BLOCK_SIZE={BLOCK_SIZE}")

Print the values at compile time. The parameters match the builtin print.

Note

Calling the Python builtin print inside a JIT function is not equivalent to static_print. The builtin maps to device_print, which emits runtime GPU print instructions with different argument requirements.

Parameters¶

*valuesvariadic

Values to print. These must be compile-time constants (e.g., constexpr or Python literals).

sepstr, optional

String inserted between values. Default is a space.

endstr, optional

String appended after the last value. Default is a newline.

fileobject, optional

Ignored. Provided for API compatibility with builtin print.

flushbool, optional

Ignored. Provided for API compatibility with builtin print.

Returns¶

None

Notes¶

This function emits output during compilation (to stdout/stderr of the compiler process), not during kernel execution. It is useful for debugging metaprogramming logic or inspecting compile-time constant values.

Examples¶

import triton
import triton.language as tl

@triton.jit
def kernel(BLOCK_SIZE: tl.constexpr):
    tl.static_print(f"BLOCK_SIZE={BLOCK_SIZE}")
    # Kernel logic here
