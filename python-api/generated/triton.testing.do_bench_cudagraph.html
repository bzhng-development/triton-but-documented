triton.testing.do_bench_cudagraph(fn, rep=20, grad_to_none=None, quantiles=None, return_mode='mean')¶

Benchmark the runtime of a CUDA function using CUDA graphs to minimize host overhead.

This function estimates the kernel execution time, constructs a CUDA graph with unrolled repetitions to reduce launch latency, and measures the graph replay time. It is suitable for benchmarking small kernels where host overhead significantly impacts measurement accuracy.

Parameters¶

fncallable

The function to benchmark. Must contain CUDA operations compatible with torch.cuda.CUDAGraph.

repint, optional

Target repetition time in milliseconds. The function determines the number of iterations within the graph to approximate this duration (default is 20).

grad_to_nonelist of torch.Tensor, optional

List of tensors whose gradients should be reset to None before each iteration. Useful when benchmarking functions involving autograd.

quantileslist of float, optional

Performance percentiles to compute (e.g., [0.2, 0.8]). If provided, overrides return_mode and returns the quantile values.

return_modestr, optional

Statistical measure to return. Options are "min", "max", "mean", "median", or "all". Default is "mean". Ignored if quantiles is specified.

Returns¶

float or list of float

Benchmark result in milliseconds. Type depends on return_mode and quantiles: - "min", "max", "mean", "median": single float. - "all": list of all measured times. - quantiles: list of quantile values (or single float if one quantile).

Notes¶

  • Uses CUDA graphs to amortize kernel launch overhead, providing more accurate measurements for fast kernels compared to do_bench.

  • Performs 5 warmup iterations before graph construction and 10 measurement retries for statistical stability.

  • Graph construction overhead is not included in the reported time.

  • Requires PyTorch with CUDA graph support.

Examples¶

import torch
import triton
import triton.testing as tt

# Define a simple CUDA kernel wrapper
def my_kernel_fn():
    # ... CUDA operations ...
    pass

# Benchmark with default settings (mean time)
mean_time = tt.do_bench_cudagraph(my_kernel_fn)

# Benchmark returning min and max times
min_time = tt.do_bench_cudagraph(my_kernel_fn, return_mode="min")
max_time = tt.do_bench_cudagraph(my_kernel_fn, return_mode="max")

# Benchmark with gradient reset for training kernels
tensor = torch.randn(1024, device='cuda', requires_grad=True)
time = tt.do_bench_cudagraph(
    lambda: my_kernel_fn(),
    grad_to_none=[tensor]
)

# Benchmark with custom quantiles (20th and 80th percentile)
q20, q80 = tt.do_bench_cudagraph(
    my_kernel_fn,
    quantiles=[0.2, 0.8]
)
