load_tensor_descriptor¶

Load a block of data from a tensor descriptor.

load_tensor_descriptor(desc, offsets, _semantic=None)

Parameters¶

desctensor_descriptor_base

The tensor descriptor object representing the global memory tensor.

offsetsSequence[constexpr | tensor]

A sequence of offsets specifying the starting coordinates for the block load. Each element corresponds to a dimension of the tensor.

_semanticoptional

Internal semantic argument; do not set manually.

Returns¶

tensor

A tensor containing the loaded data with shape defined by the descriptor’s block shape.

Notes¶

This function is a wrapper around the tensor_descriptor_base.load() method. It requires the descriptor to be created via triton.language.make_tensor_descriptor(). Offsets must align with the hardware requirements (typically 16-byte alignment for TMA). Out-of-bounds access behavior is determined by the descriptor’s padding option (default is zero).

Examples¶

import triton
import triton.language as tl

@triton.jit
def kernel(desc_ptr, M, N, BLOCK_M: tl.constexpr, BLOCK_N: tl.constexpr):
    # Create a tensor descriptor for a 2D tensor
    desc = tl.make_tensor_descriptor(
        base=desc_ptr,
        shape=[M, N],
        strides=[N, 1],
        block_shape=[BLOCK_M, BLOCK_N],
    )

    # Calculate offsets based on program ID
    offset_m = tl.program_id(0) * BLOCK_M
    offset_n = tl.program_id(1) * BLOCK_N

    # Load a block of data using the descriptor
    data = tl.load_tensor_descriptor(desc, [offset_m, offset_n])

    # Perform computation
    res = tl.abs(data)

    # Store result back using the descriptor
    tl.store_tensor_descriptor(desc, [offset_m, offset_n], res)
