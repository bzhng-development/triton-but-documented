Triton

Compute the matrix product of two 2D or 3D tensors.

For 3D inputs, performs a batched matrix multiplication where the first dimension represents the batch. Supports optional accumulation and precision control for Tensor Core operations.

Parameters¶

inputtensor

The first tensor to be multiplied. Must be 2D or 3D. Supported scalar types: int8, float8_e5m2, float16, bfloat16, float32.

othertensor

The second tensor to be multiplied. Must be 2D or 3D with compatible inner dimensions. Supported scalar types: int8, float8_e5m2, float16, bfloat16, float32.

acctensor, optional

Accumulator tensor. If provided, the result is added to this tensor before returning. Must match the output shape. Supported types: float16, float32, int32.

input_precisionstr, optional

Precision mode for float32 x float32 operations on Tensor Cores. Ignored if inputs are not float32 or hardware lacks Tensor Cores. Options for NVIDIA: "tf32", "tf32x3", "ieee" (default: "tf32"). Options for AMD: "ieee", "tf32" (CDNA3 only).

allow_tf32bool, optional

Deprecated. If True, sets input_precision="tf32". Mutually exclusive with input_precision; at least one must be None.

max_num_imprecise_accint, optional

Maximum number of imprecise accumulations allowed (hardware-specific).

out_dtypedtype, optional

Data type of the output tensor. Default: tl.float32.

Returns¶

tensor

The matrix product of input and other (plus acc if provided), with shape [..., M, N] where input is [..., M, K] and other is [..., K, N].

Notes¶

When using TF32 precision, float32 inputs may be truncated to TF32 format (19-bit floating point) without rounding, which can introduce bias. For best results, explicitly round inputs to TF32 or load data using TensorDescriptor with round_f32_to_tf32=True.

Examples¶

import triton
import triton.language as tl

@triton.jit
def matmul_kernel(
    a_ptr, b_ptr, c_ptr,
    M, N, K,
    BLOCK_SIZE_M: tl.constexpr, BLOCK_SIZE_N: tl.constexpr, BLOCK_SIZE_K: tl.constexpr,
):
    pid = tl.program_id(axis=0)
    num_pid_m = tl.cdiv(M, BLOCK_SIZE_M)
    pid_m = pid % num_pid_m
    pid_n = pid // num_pid_m

    offs_m = (pid_m * BLOCK_SIZE_M + tl.arange(0, BLOCK_SIZE_M)) % M
    offs_n = (pid_n * BLOCK_SIZE_N + tl.arange(0, BLOCK_SIZE_N)) % N
    offs_k = tl.arange(0, BLOCK_SIZE_K)

    a_ptrs = a_ptr + offs_m[:, None] * K + offs_k[None, :]
    b_ptrs = b_ptr + offs_k[:, None] * N + offs_n[None, :]

    accumulator = tl.zeros((BLOCK_SIZE_M, BLOCK_SIZE_N), dtype=tl.float32)
    for k in range(0, K, BLOCK_SIZE_K):
        a = tl.load(a_ptrs, mask=(offs_m[:, None] < M) & (offs_k[None, :] < K - k), other=0.0)
        b = tl.load(b_ptrs, mask=(offs_k[:, None] < K - k) & (offs_n[None, :] < N), other=0.0)
        accumulator = tl.dot(a, b, acc=accumulator, input_precision="tf32")
        a_ptrs += BLOCK_SIZE_K
        b_ptrs += BLOCK_SIZE_K * N

    c_ptrs = c_ptr + offs_m[:, None] * N + offs_n[None, :]
    tl.store(c_ptrs, accumulator, mask=(offs_m[:, None] < M) & (offs_n[None, :] < N))
