set_allocator¶

triton.set_allocator(allocator: Allocator) → None¶

The allocator function is called during kernel launch for kernels that require additional global memory workspace.

Sets the global memory allocator used by Triton kernels during launch.

Some Triton kernels require additional global memory workspace for temporary buffers (e.g., for atomic operations, sorting, or reduction). This function registers a custom allocator that Triton will invoke to reserve such memory before kernel execution. If no allocator is set, kernels requiring dynamic allocation will raise a RuntimeError.

Parameters¶

allocatortriton.runtime._allocation.Allocator

A callable object conforming to the Allocator protocol. It must accept three arguments:

  • size (int): The number of bytes to allocate.

  • alignment (int): The required memory alignment in bytes.

  • stream (Optional[int]): The CUDA stream ID (or None for the default stream).

The callable must return an object conforming to the Buffer protocol, which provides a data_ptr() method returning the device memory address as an integer.

Returns¶

None

Notes¶

The allocator is stored in a context variable and is consulted every time a kernel requiring dynamic global memory is launched. The default allocator raises an error if invoked; users must explicitly call set_allocator to enable kernels with dynamic memory requirements.

The allocator is typically implemented using framework-specific memory allocation APIs (e.g., torch.cuda.memory.alloc for PyTorch or jax.dlpack.to_device for JAX).

Examples¶

import triton
import triton.language as tl
import torch

# Define a custom allocator using PyTorch's CUDA memory API
def torch_allocator(size: int, alignment: int, stream=None):
    # Allocate tensor on CUDA device
    tensor = torch.empty(size, dtype=torch.uint8, device='cuda')
    # Ensure alignment if needed (PyTorch tensors are usually well-aligned)
    return tensor

# Register the allocator with Triton
triton.set_allocator(torch_allocator)

# Now kernels requiring dynamic memory can be launched successfully
@triton.jit
def kernel_with_dynamic_mem(...):
    # Kernel logic using dynamic shared/global memory
    ...
