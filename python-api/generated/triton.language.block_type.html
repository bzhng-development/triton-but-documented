block_type¶

class triton.language.block_type(element_ty, shape)¶

Represents a contiguous block of scalar elements in GPU memory.

block_type is the underlying type for tensor objects when the tensor has a known, static shape. It defines the element data type and the multidimensional shape of the block.

Parameters¶

element_tydtype

The scalar data type of the elements (e.g., tl.float16, tl.int32).

shapelist or tuple of ints

The dimensions of the block. Must be a non-empty sequence of integers. A 0-dimensional shape is forbidden.

Attributes¶

element_tydtype

The scalar data type of the elements.

shapetuple of int

The dimensions of the block.

numelint

The total number of elements in the block (product of shape dimensions).

nbytesint

The total size in bytes of the block.

scalardtype

Alias for element_ty.

Methods¶

get_block_shapes()

Returns the shape of the block as a tuple of integers.

with_element_ty(scalar_ty)

Returns a new block_type with the same shape but a different element type.

to_ir(builder)

Converts the type to the corresponding MLIR IR type.

is_block()

Returns True (identifies this type as a block type).

mangle()

Returns a mangled string representation for name mangling.

Notes¶

block_type is typically not instantiated directly by users. Instead, tensors with specific shapes are created using operations like tl.full(), tl.arange(), or tl.make_block_ptr(), which infer the block_type automatically.

The shape argument for block_type uses Python int values, whereas the shape attribute of a tensor object contains constexpr values.

Examples¶

block_type is implicitly used when creating typed tensors:

import triton
import triton.language as tl

@triton.jit
def kernel(
    BLOCK_M: tl.constexpr, BLOCK_N: tl.constexpr
):
    # Create a 2D tensor of float16.
    # The underlying type is block_type(element_ty=fp16, shape=[BLOCK_M, BLOCK_N])
    block = tl.full([BLOCK_M, BLOCK_N], 0.0, dtype=tl.float16)

    # Accessing the type properties (mostly internal/advanced usage)
    # In a JIT context, direct inspection of types is limited,
    # but conceptually:
    # block.type == tl.block_type(tl.float16, [BLOCK_M, BLOCK_N])
    pass
