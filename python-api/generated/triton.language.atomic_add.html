atomic_add¶

triton.language.atomic_add(pointer, val, mask=None, sem=None, scope=None)¶

Performs an atomic add at the memory location specified by pointer.

Returns the data stored at pointer before the atomic operation.

This function can also be called as a member function on tensor, as x.atomic_add(...) instead of atomic_add(x, ...).

Parameters¶

pointertriton.PointerType, or block of dtype=triton.PointerType

The memory locations to operate on.

valBlock

The values with which to perform the atomic operation. Must have dtype equal to pointer.dtype.element_ty.

maskBlock of triton.int1, optional

If mask[idx] is false, do not perform the atomic operation at pointer[idx].

semstr, optional

Specifies the memory semantics for the operation. Acceptable values are “acquire”, “release”, “acq_rel” (ACQUIRE_RELEASE), and “relaxed”. Defaults to “acq_rel”.

scopestr, optional

Defines the scope of threads that observe the synchronizing effect of the atomic operation. Acceptable values are “gpu” (default), “cta” (cooperative thread array), or “sys” (SYSTEM).

Returns¶

Block

A tensor containing the values at pointer before the atomic operation was performed. The shape and dtype match val.

Example¶

@triton.jit
def atomic_add_kernel(x_ptr, y_ptr, BLOCK_SIZE: tl.constexpr):
    pid = tl.program_id(0)
    offsets = pid * BLOCK_SIZE + tl.arange(0, BLOCK_SIZE)
    x = tl.load(x_ptr + offsets)
    # Atomically add x to y, returning the old value in y_old
    y_old = tl.atomic_add(y_ptr + offsets, x)
    tl.store(x_ptr + offsets, y_old)
