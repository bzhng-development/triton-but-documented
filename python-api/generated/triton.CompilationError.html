CompilationError¶

Base class for all errors raised during Triton kernel compilation.

This exception provides detailed diagnostic information, including the source code excerpt surrounding the error location and a pointer to the specific column where the issue occurred. It is designed to facilitate debugging of JIT-compiled GPU kernels.

Parameters¶

srcstr, optional

The source code string of the kernel being compiled. If None, the error message will indicate that the source is unavailable.

nodeast.AST

The Python Abstract Syntax Tree (AST) node where the compilation error occurred. This node must contain lineno and col_offset attributes for precise location reporting.

error_messagestr, optional

A specific description of the compilation failure. If provided, this message is appended to the source excerpt in the final error output.

Notes¶

The error message generated by this class includes up to 12 lines of source code preceding the error location, followed by a caret (^) marking the exact column offset. This formatting aids in quickly identifying syntax or semantic issues in kernel definitions.

The class implements __reduce__ to ensure instances are picklable, allowing them to be serialized when using multiprocessing or distributed compilation strategies.

Examples¶

import ast
import triton
from triton.compiler.errors import CompilationError

# Simulate a kernel source with an error
src = """
@triton.jit
def kernel_fn(ptr):
    # Invalid operation
    let x = 1
"""

# Parse the source to get an AST node (typically done internally by the compiler)
tree = ast.parse(src)
error_node = tree.body[0].body[1]  # Point to the line with the error

# Raise a compilation error with context
try:
    raise CompilationError(src=src, node=error_node, error_message="Invalid Python syntax in kernel")
except CompilationError as e:
    print(e)
    # Output includes the source snippet and caret pointer:
    # at 4:8:
    #     # Invalid operation
    #     let x = 1
    #         ^
    # Invalid Python syntax in kernel
