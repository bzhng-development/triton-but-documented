reinterpret¶

triton.reinterpret(tensor, dtype)¶

Create a view of a tensor with a different data type without copying the underlying data.

This function returns a TensorWrapper that interprets the same memory buffer as the specified dtype. It is useful for type-punning operations in GPU kernels where the bit pattern of data needs to be reinterpreted as a different type (e.g., viewing float32 bits as int32).

Parameters¶

tensortensor-like

The input tensor to reinterpret. Must have a data_ptr() method or be an existing TensorWrapper.

dtypedtype

The target data type for the reinterpretation.

Returns¶

TensorWrapper or tensor

A TensorWrapper object viewing the original data as dtype. If tensor is already a TensorWrapper and dtype matches its base type, the base tensor is returned directly.

Raises¶

TypeError

If tensor does not have a data_ptr() method and is not a TensorWrapper.

Notes¶

This operation does not perform any data conversion or memory movement; it only changes how the existing bits are interpreted. Ensure that the size of the original dtype and the target dtype are compatible with your intended access patterns to avoid out-of-bounds memory errors.

Examples¶

import triton
import triton.language as tl

@triton.jit
def kernel(x_ptr, y_ptr, n):
    # Load float32 data
    fp_val = tl.load(x_ptr)

    # Reinterpret bits as int32
    int_val = tl.load(tl.reinterpret(x_ptr, tl.int32))

    # Store result
    tl.store(y_ptr, int_val)

# Usage note: reinterpret is often used on pointers or tensors
# before passing to load/store operations in kernels.
