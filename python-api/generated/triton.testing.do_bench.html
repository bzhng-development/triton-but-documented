triton.testing.do_bench(fn, warmup=25, rep=100, grad_to_none=None, quantiles=None, return_mode='mean')¶

Benchmark the runtime of the provided function.

By default, returns the median runtime of fn along with the 20th and 80th performance percentiles if quantiles is not specified. The function automatically estimates the kernel execution time to determine the number of warmup and repetition iterations required to meet the warmup and rep duration targets (in milliseconds). It clears the L2 cache before each repetition to ensure consistent benchmarking conditions.

Parameters¶

fncallable

Function to benchmark. Typically a Triton kernel launch or a callable containing GPU operations.

warmupint, optional

Total warmup time in milliseconds. The number of warmup iterations is calculated based on the estimated kernel runtime. Default is 25.

repint, optional

Total repetition time in milliseconds for the actual benchmarking phase. The number of repetitions is calculated based on the estimated kernel runtime. Default is 100.

grad_to_nonelist of torch.Tensor, optional

List of tensors whose gradients should be reset to None before each repetition. This prevents gradient accumulation if fn includes backward passes.

quantileslist of float, optional

List of quantiles to compute (e.g., [0.2, 0.8]). If provided, returns the computed quantile values instead of the statistic defined by return_mode.

return_modestr, optional

The statistical measure to return when quantiles is None. Options are: "min", "max", "mean", "median", or "all" (returns the full list of measured times). Default is "mean".

Returns¶

float or list of float

The benchmark result. If quantiles is provided, returns the quantile values (scalar if single quantile, list otherwise). If return_mode is "all", returns a list of all measured execution times in milliseconds. Otherwise, returns the specified statistical measure (float).

Notes¶

The function performs an initial dry run to estimate kernel latency, then dynamically computes the number of iterations needed to satisfy the warmup and rep time constraints. L2 cache is flushed before every timed iteration to minimize interference from cache hits.

Examples¶

>>> import triton
>>> import triton.language as tl
>>> import torch
>>>
>>> @triton.jit
>>> def kernel_fn(X, Y, BLOCK_SIZE: tl.constexpr):
...     pid = tl.program_id(0)
...     x = tl.load(X + pid * BLOCK_SIZE)
...     tl.store(Y + pid * BLOCK_SIZE, x * 2)
>>>
>>> x = torch.ones((1024,), device="cuda")
>>> y = torch.zeros((1024,), device="cuda")
>>>
>>> # Benchmark with default settings (mean time)
>>> mean_time = triton.testing.do_bench(lambda: kernel_fn[(1,)](x, y, 1024))
>>>
>>> # Benchmark returning min, max, and median
>>> min_time = triton.testing.do_bench(lambda: kernel_fn[(1,)](x, y, 1024), return_mode="min")
>>>
>>> # Benchmark with custom quantiles (20th and 80th percentile)
>>> p20, p80 = triton.testing.do_bench(
...     lambda: kernel_fn[(1,)](x, y, 1024),
...     quantiles=[0.2, 0.8]
... )
