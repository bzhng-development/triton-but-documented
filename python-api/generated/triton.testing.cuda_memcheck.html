cuda_memcheck¶

triton.testing.cuda_memcheck(**target_kwargs)¶

Decorator that runs a pytest test function under cuda-memcheck to detect out-of-bounds memory accesses and other CUDA runtime errors.

The decorator inspects the calling process name and test arguments. If the parent process is not already cuda-memcheck and the provided target_kwargs are present in the test call, it re-executes the specific test node via subprocess with cuda-memcheck enabled. It asserts that the execution returns successfully with zero errors.

Parameters¶

**target_kwargsdict

Keyword arguments used to conditionally trigger the memory check. The decorator only invokes cuda-memcheck if these items are a subset of the arguments passed to the wrapped test function.

Returns¶

decoratorcallable

A decorator that wraps the target test function with memory checking logic.

Raises¶

AssertionError

If the wrapped test function does not include a request fixture (required for identifying the test node), or if cuda-memcheck reports any errors or non-zero return code.

Notes¶

  • The decorated test function must accept a request fixture (typically provided by pytest), even if it is unused within the test body.

  • Sets the environment variable PYTORCH_NO_CUDA_MEMORY_CACHING=1 during the memcheck run to ensure accurate memory tracking.

  • This decorator is intended for use in CI or debugging workflows where detecting illegal memory access is critical.

Examples¶

import triton
import triton.testing as tt
import pytest

@tt.cuda_memcheck(mode="strict")
def test_kernel_bounds(request):
    # Define a kernel that may have out-of-bounds access
    @triton.jit
    def kernel(x_ptr, n):
        pid = triton.program_id(0)
        if pid < n:
            val = triton.load(x_ptr + pid)
            triton.store(x_ptr + pid, val + 1)

    # Launch kernel
    x = torch.zeros(1024, device='cuda')
    kernel[(1024,)](x, 1024)

# Run with pytest: the decorator will invoke cuda-memcheck automatically
# if 'mode="strict"' is passed.
